Отличная задача! **CartPole** (тележка с шестом) — это классическая среда для знакомства с обучением с подкреплением (Reinforcement Learning). 

Самый популярный и понятный алгоритм, использующий нейросети для этой задачи — **DQN (Deep Q-Network)**. Мы напишем его с использованием современных библиотек: **Gymnasium** (официальный преемник OpenAI Gym) и **PyTorch** (для нейросетей).

### Полный код решения
Ниже представлен полностью рабочий код. Я снабдил его подробными комментариями.

```python
import gymnasium as gym
import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np
import random
from collections import deque

# 1. Создаем саму нейронную сеть
class QNetwork(nn.Module):
    def __init__(self, input_dim, output_dim):
        super(QNetwork, self).__init__()
        # Для CartPole достаточно простой полносвязной сети из 3 слоев
        self.network = nn.Sequential(
            nn.Linear(input_dim, 128),
            nn.ReLU(),
            nn.Linear(128, 128),
            nn.ReLU(),
            nn.Linear(128, output_dim)
        )

    def forward(self, x):
        return self.network(x)

# 2. Создаем Агента, который будет принимать решения и обучаться
class DQNAgent:
    def __init__(self, state_dim, action_dim):
        self.state_dim = state_dim
        self.action_dim = action_dim
        
        # Гиперпараметры
        self.gamma = 0.99           # Скидка будущих наград (насколько важны будущие шаги)
        self.epsilon = 1.0          # Вероятность случайного действия (исследование)
        self.epsilon_min = 0.01     # Минимальная вероятность случайного действия
        self.epsilon_decay = 0.995  # Как быстро агент перестает исследовать
        self.learning_rate = 0.001
        self.batch_size = 64
        
        # Буфер памяти для обучения на прошлом опыте (Experience Replay)
        self.memory = deque(maxlen=10000)
        
        # Инициализация нейросети и оптимизатора
        self.model = QNetwork(state_dim, action_dim)
        self.optimizer = optim.Adam(self.model.parameters(), lr=self.learning_rate)
        self.loss_fn = nn.MSELoss() # Среднеквадратичная ошибка

    def act(self, state):
        # Epsilon-жадная стратегия: иногда делаем случайный шаг
        if random.random() < self.epsilon:
            return random.randint(0, self.action_dim - 1)
        
        # Иначе просим нейросеть предсказать лучшее действие
        state_tensor = torch.FloatTensor(state).unsqueeze(0)
        with torch.no_grad():
            q_values = self.model(state_tensor)
        return q_values.argmax().item()

    def remember(self, state, action, reward, next_state, done):
        # Запоминаем шаг
        self.memory.append((state, action, reward, next_state, done))

    def train(self):
        # Если накопилось мало опыта — не обучаемся
        if len(self.memory) < self.batch_size:
            return

        # Берем случайную выборку (батч) из памяти
        batch = random.sample(self.memory, self.batch_size)
        states, actions, rewards, next_states, dones = zip(*batch)

        # Конвертируем в тензоры PyTorch
        states = torch.FloatTensor(np.array(states))
        actions = torch.LongTensor(actions).unsqueeze(1)
        rewards = torch.FloatTensor(rewards).unsqueeze(1)
        next_states = torch.FloatTensor(np.array(next_states))
        dones = torch.FloatTensor(dones).unsqueeze(1)

        # Q-значения, которые сеть предсказывает для текущих состояний
        current_q = self.model(states).gather(1, actions)

        # Максимальные Q-значения для следующих состояний
        with torch.no_grad():
            max_next_q = self.model(next_states).max(1)[0].unsqueeze(1)
            # Уравнение Беллмана
            target_q = rewards + (1 - dones) * self.gamma * max_next_q

        # Считаем ошибку и обновляем веса нейросети
        loss = self.loss_fn(current_q, target_q)
        
        self.optimizer.zero_grad()
        loss.backward()
        self.optimizer.step()

        # Уменьшаем epsilon (агент становится более "уверенным")
        if self.epsilon > self.epsilon_min:
            self.epsilon *= self.epsilon_decay

# 3. Основной цикл обучения
def main():
    # Создаем среду
    env = gym.make('CartPole-v1')
    
    state_dim = env.observation_space.shape[0] # 4 параметра (координаты, скорости, углы)
    action_dim = env.action_space.n            # 2 действия (влево, вправо)
    
    agent = DQNAgent(state_dim, action_dim)
    episodes = 500

    for episode in range(episodes):
        state, info = env.reset()
        total_reward = 0
        done = False
        truncated = False

        while not (done or truncated):
            # Агент выбирает действие
            action = agent.act(state)
            
            # Среда реагирует на действие
            next_state, reward, done, truncated, info = env.step(action)
            
            # Запоминаем результат и обучаемся
            agent.remember(state, action, reward, next_state, done)
            agent.train()
            
            state = next_state
            total_reward += reward

        print(f"Эпизод: {episode + 1}/{episodes} | Награда (Очки): {total_reward} | Epsilon: {agent.epsilon:.2f}")

        # Среда считается решенной, если агент удерживает шест 500 шагов
        if total_reward >= 500:
            print(f"Среда успешно решена за {episode + 1} эпизодов!")
            break

    env.close()
    
    # 4. Демонстрация обученного агента
    print("Запускаем демонстрацию...")
    env_eval = gym.make('CartPole-v1', render_mode='human')
    state, info = env_eval.reset()
    agent.epsilon = 0.0 # Отключаем случайные действия
    
    while True:
        action = agent.act(state)
        state, reward, done, truncated, info = env_eval.step(action)
        if done or truncated:
            break
    env_eval.close()

if __name__ == "__main__":
    main()
```

### Как это работает (кратко):

1. **Состояние (State):** Массив из 4 чисел (позиция тележки, скорость тележки, угол наклона шеста, скорость падения шеста). Нейросеть принимает эти 4 числа на вход.
2. **Действие (Action):** Массив из 2 чисел (Q-значения для шага «влево» и шага «вправо»). Нейросеть выдает эти 2 числа на выходе. Агент выбирает то действие, у которого число больше.
3. **Experience Replay (Буфер памяти):** Нейросети плохо обучаются на последовательных данных, так как они сильно связаны друг с другом. Поэтому агент складывает каждый свой шаг в «память», а потом достает случайную горсть (batch) из 64 шагов и учится на них.
4. **Уравнение Беллмана:** Сеть учится предсказывать не просто награду прямо сейчас, а сумму всех будущих наград.
5. **Epsilon ($\epsilon$):** В начале обучения агент ничего не знает и делает случайные шаги (Epsilon = 1.0). С каждым шагом этот показатель уменьшается, и агент начинает полагаться на предсказания нейросети.

Обычно этот код решает CartPole (набирает максимальные 500 очков) за **100–250 эпизодов**. В конце обучения появится окно, где вы сможете своими глазами увидеть, как тележка идеально балансирует шест!

Теперь давай напишем модель для того же окружения, но используя библиотеку neat-python

Отличный выбор! Использование **NEAT** (NeuroEvolution of Augmenting Topologies) кардинально меняет подход к задаче. 

Если в DQN (из предыдущего примера) мы брали готовую нейросеть фиксированного размера и с помощью сложной математики (градиентного спуска) подгоняли её веса, то **NEAT использует генетический алгоритм (эволюцию)**. 

Алгоритм создает "популяцию" случайных крошечных нейросетей, проверяет их в игре, отбраковывает слабые, а сильные скрещивает и мутирует (добавляет новые нейроны и связи).

Установи библиотеку `neat-python`:


### Специфика NEAT
Для `neat-python` **обязательно** нужен конфигурационный текстовый файл, в котором описаны параметры эволюции (размер популяции, шансы мутаций и т.д.). Чтобы тебе не пришлось создавать его вручную, мой код **сам создаст этот файл** перед запуском.

### Полный код решения (NEAT + CartPole)

```python
import gymnasium as gym
import neat
import os
import numpy as np

# 1. Текст конфигурационного файла для NEAT
# Специфично для CartPole: 4 входа (состояние), 2 выхода (действия).
CONFIG_TEXT = """
[NEAT]
fitness_criterion     = max
fitness_threshold     = 500
pop_size              = 50
reset_on_extinction   = False

[DefaultGenome]
# Настройки топологии сети
num_inputs              = 4
num_outputs             = 2
num_hidden              = 0
initial_connection      = full_direct
feed_forward            = True

# Настройки активационных функций
activation_default      = relu
activation_mutate_rate  = 0.1
activation_options      = relu

# Настройки агрегации узлов
aggregation_default     = sum
aggregation_mutate_rate = 0.0
aggregation_options     = sum

# Настройки весов связей
bias_init_mean          = 0.0
bias_init_stdev         = 1.0
bias_max_value          = 30.0
bias_min_value          = -30.0
bias_mutate_power       = 0.5
bias_mutate_rate        = 0.7
bias_replace_rate       = 0.1

weight_init_mean        = 0.0
weight_init_stdev       = 1.0
weight_max_value        = 30
weight_min_value        = -30
weight_mutate_power     = 0.5
weight_mutate_rate      = 0.8
weight_replace_rate     = 0.1

# Настройки мутаций топологии (добавление узлов и связей)
conn_add_prob           = 0.5
conn_delete_prob        = 0.5
node_add_prob           = 0.2
node_delete_prob        = 0.2

# Скрещивание
compatibility_disjoint_coefficient = 1.0
compatibility_weight_coefficient   = 0.5

[DefaultSpeciesSet]
compatibility_threshold = 3.0

[DefaultStagnation]
species_fitness_func = max
max_stagnation       = 20
species_elitism      = 2

[DefaultReproduction]
elitism            = 2
survival_threshold = 0.2
"""

# 2. Функция для создания конфиг файла (чтобы было удобно запускать один скрипт)
def create_config_file(filename="config-neat.txt"):
    with open(filename, "w") as f:
        f.write(CONFIG_TEXT)
    return filename

# 3. Фитнес-функция (Оценка выживаемости)
def eval_genomes(genomes, config):
    """
    Эта функция запускается для КАЖДОГО поколения.
    Она берет каждого "генома" (нейросеть) из популяции, играет им в игру
    и присваивает ему очки (fitness) за то, как долго он продержался.
    """
    env = gym.make('CartPole-v1')
    
    for genome_id, genome in genomes:
        # Создаем нейросеть прямого распространения из генома
        net = neat.nn.FeedForwardNetwork.create(genome, config)
        
        state, info = env.reset()
        fitness = 0.0
        done = False
        truncated = False
        
        while not (done or truncated):
            # Передаем 4 параметра среды в нейросеть. 
            # На выходе получаем 2 числа (уверенность в шаге влево и вправо)
            output = net.activate(state)
            
            # Выбираем действие с максимальной уверенностью (0 или 1)
            action = np.argmax(output)
            
            # Делаем шаг в среде
            state, reward, done, truncated, info = env.step(action)
            fitness += reward
            
        # Записываем результат генома (сколько шагов он удержал палку)
        genome.fitness = fitness
        
    env.close()

# 4. Основной цикл эволюции
def run_neat():
    # Создаем файл настроек
    config_path = create_config_file()
    
    # Загружаем настройки
    config = neat.Config(neat.DefaultGenome, neat.DefaultReproduction,
                         neat.DefaultSpeciesSet, neat.DefaultStagnation,
                         config_path)

    # Создаем популяцию
    p = neat.Population(config)

    # Добавляем репортеры, чтобы видеть в консоли статистику по поколениям (Generations)
    p.add_reporter(neat.StdOutReporter(True))
    stats = neat.StatisticsReporter()
    p.add_reporter(stats)

    print("Начинаем эволюцию...")
    # Запускаем эволюцию! Максимум 50 поколений.
    # Если кто-то наберет 500 очков (fitness_threshold в конфиге), эволюция остановится досрочно.
    winner = p.run(eval_genomes, 50)

    print(f'\nЭволюция завершена! Лучший геном: {winner}')

    # 5. Демонстрация лучшего генома-победителя
    print("\nЗапускаем демонстрацию лучшего агента...")
    env_eval = gym.make('CartPole-v1', render_mode='human')
    winner_net = neat.nn.FeedForwardNetwork.create(winner, config)
    
    state, info = env_eval.reset()
    done = False
    truncated = False
    
    while not (done or truncated):
        output = winner_net.activate(state)
        action = np.argmax(output)
        state, reward, done, truncated, info = env_eval.step(action)
        
    env_eval.close()

if __name__ == '__main__':
    run_neat()
```

### Разница с DQN — как это работает тут:

1. **Нет Backpropagation:** Мы не считаем никаких ошибок (`Loss`) и не используем оптимизаторы вроде Adam. Мы вообще не меняем веса нейросети во время ее "жизни". 
2. **Поколения (Generations):** За один раз создается 50 сетей (`pop_size = 50`). Они все играют в игру. Те, кто уронил палку сразу — удаляются. Те, кто продержал долго — скрещиваются между собой (их веса и связи перемешиваются).
3. **Рост из ниоткуда:** Обрати внимание в конфигурации: `num_hidden = 0`. Нейросети рождаются вообще без скрытых слоев. Входы напрямую соединены с выходами. С каждым поколением алгоритм *сам* решает, добавить ли новый промежуточный нейрон или новую связь, если это повышает выживаемость.
4. **Виды (Species):** NEAT защищает инновации. Если рождается сеть с новой причудливой структурой, она поначалу может играть плохо. NEAT выделяет её в отдельный "вид", давая время на оптимизацию весов внутри своей группы, чтобы другие, более сильные виды, не уничтожили её сразу.

Для задачи CartPole **NEAT является избыточно мощным**. Зачастую он находит решение (сеть, набирающую 500 очков) уже на **1–5 поколении**, в то время как DQN требует сотни эпизодов. При этом итоговая нейросеть у NEAT получается невероятно крошечной и быстрой по сравнению с огромными матрицами в PyTorch.

### Часть 1. Как работает алгоритм NEAT (на пальцах)

**NEAT (NeuroEvolution of Augmenting Topologies)** работает точно так же, как эволюция в природе. Он не использует математический расчет ошибок (как градиентный спуск в обычных нейросетях). Он использует **естественный отбор**.

Процесс идет по кругу:
1. **Поколение 0 (Создание).** Создается 50 (размер популяции) абсолютно глупых "агентов". У них нет скрытых слоев нейронов — только 4 входа напрямую связаны с 2 выходами. Веса (сила связей) заданы случайно.
2. **Экзамен (Fitness).** Каждый агент по очереди пытается удержать палку (CartPole). За каждый кадр удержания он получает 1 очко. Кто-то роняет палку на 10-м шаге (оценка 10), кто-то на 30-м (оценка 30).
3. **Естественный отбор.** Худшие 80% агентов безжалостно удаляются (умирают). 
4. **Размножение (Скрещивание).** Лучшие 20% агентов "спариваются" друг с другом. Берутся куски нейросети "мамы" и "папы", чтобы создать ребенка.
5. **Мутации (Магия NEAT).** У детей могут произойти мутации:
   * *Мутация весов:* Цифра, отвечающая за силу связи, немного меняется.
   * *Мутация топологии:* Внезапно появляется **новая связь** между нейронами или вырастает **абсолютно новый нейрон** (скрытый слой). Мозг усложняется!
6. **Разделение на Виды (Speciation).** Если у агента мутировал новый нейрон, поначалу он может работать плохо. Чтобы его сразу не убил естественный отбор, NEAT выделяет таких "мутантов" в отдельный **Вид**. Они конкурируют только внутри своего вида, пока не разовьют потенциал своей новой структуры мозга.
7. **Поколение 1, 2, 3...** Процесс повторяется, пока кто-то не наберет 500 очков.

### Часть 2. Разбор твоего конфигурационного файла

#### Секция `[NEAT]` (Общие правила мира)
* `fitness_criterion = max` — Цель эволюции: максимизировать очки.
* `fitness_threshold = 500` — Условие победы. Если агент набрал 500 очков, останавливаем эволюцию.
* `pop_size = 50` — Размер популяции (сколько агентов рождается в одном поколении).
* `reset_on_extinction = False` — Если все виды вымерли, не начинать с нуля (просто защита от крашей).
* `no_fitness_termination = False` — Разрешить алгоритму остановиться, если достигнут `fitness_threshold`.

#### Секция `[DefaultGenome]` (ДНК нейросети)

**1. Твои TODO (Фиксы поведения библиотеки)**
* `single_structural_mutation = false` — Разрешено ли за одно поколение делать сразу несколько структурных мутаций (и новый узел, и новую связь). `False` — да, разрешено делать много мутаций сразу.
* `structural_mutation_surer = default` — Способ распределения мутаций, оставлен по умолчанию.

**2. Базовая форма (при рождении)**
* `num_inputs = 4` — 4 параметра от игры (позиция, угол и т.д.).
* `num_outputs = 2` — 2 кнопки (влево, вправо).
* `num_hidden = 0` — Изначально скрытых слоев нет! Мозг простейший.
* `initial_connection = full_direct` — При рождении каждый из 4 входов соединен с каждым из 2 выходов (итого 8 связей).
* `feed_forward = True` — Сигнал идет только вперед (от входа к выходу). Нет рекуррентности (памяти о прошлых кадрах).

**3. Как работают сами нейроны**
* `activation... = relu` — Функция активации. `ReLU` пропускает положительные сигналы как есть, а отрицательные превращает в ноль. Мутировать эту функцию мы запрещаем (`rate = 0.1` — почти не мутирует, а `options = relu` значит, что даже если мутирует, то только в `relu`).
* `aggregation... = sum` — Если в один нейрон приходят сигналы от трех других, он их просто складывает (`sum`).

**4. Настройки Весов (`weight`) и Смещений (`bias`)**
*Это математические коэффициенты (множители), которые висят на связях между нейронами. Именно они определяют, "важен" этот сигнал или нет.*
* `_init_type = gaussian` / `_mean = 0.0` / `_stdev = 1.0` — При создании новых связей их вес генерируется случайно по Гауссову распределению (колокол) с центром в нуле.
* `_max_value = 30` / `_min_value = -30` — Ограничители. Вес не может улететь в бесконечность, иначе математика сломается.
* `_mutate_rate = 0.8` — **Вероятность 80%**, что при рождении ребенка вес связи немного изменится.
* `_mutate_power = 0.5` — Насколько сильно изменится вес при мутации (сила толчка).
* `_replace_rate = 0.1` — **Вероятность 10%**, что старый вес просто удалится и заменится абсолютно случайным новым числом.

**5. Структурные мутации (Отращивание новых извилин)**
* `conn_add_prob = 0.5` — Вероятность 50%, что у ребенка появится новая связь между случайными нейронами.
* `conn_delete_prob = 0.5` — Вероятность 50%, что какая-то связь навсегда порвется.
* `node_add_prob = 0.2` — Вероятность 20%, что посреди связи вырастет **новый нейрон** (скрытый слой).
* `node_delete_prob = 0.2` — Вероятность 20% удалить нейрон.

**6. Активность связей (`enabled`)**
* `enabled_default = True` — Все новые связи сразу работают.
* `enabled_mutate_rate = 0.05` — Вероятность 5%, что связь "уснет" (не будет пропускать сигнал, как аппендикс в организме) или наоборот "проснется".

**7. Разделение на виды (Скрещивание)**
*Как понять, что две нейросети — это разные виды животных и скрещивать их нельзя? Высчитывается "генетическое расстояние".*
* `compatibility_disjoint_coefficient = 1.0` — Насколько сильно влияет разница в **структуре** (у одного есть нейрон, у другого нет).
* `compatibility_weight_coefficient = 0.5` — Насколько сильно влияет разница в **цифрах** (весах) при одинаковой структуре.

#### Секция `[DefaultSpeciesSet]` (Правила Видов)
* `compatibility_threshold = 3.0` — Порог. Если генетическое расстояние между двумя нейросетями больше `3.0`, они считаются разными Видами (например, рептилиями и млекопитающими) и больше не спариваются друг с другом.

#### Секция `[DefaultStagnation]` (Защита от тупика эволюции)
* `max_stagnation = 20` — Если какой-то Вид живет уже 20 поколений, но его максимальный рекорд не растет (застрял в тупике эволюции), весь этот Вид вымирает (удаляется алгоритмом), освобождая место для других.
* `species_elitism = 2` — Защитить 2 самых лучших Вида от вымирания из-за стагнации (вдруг они всё-таки найдут решение).

#### Секция `[DefaultReproduction]` (Как делаются дети)
* `survival_threshold = 0.2` — Только Топ-20% лучших представителей вида имеют право размножаться. Остальные 80% умирают без потомства.
* `elitism = 2` — Две самые лучшие нейросети в виде переходят в следующее поколение **вообще без изменений и мутаций**. Это гарантирует, что мы случайно не испортим мутациями нашу лучшую находку.

### Итог
Этот конфиг заставляет агентов начать с абсолютного нуля (без скрытых нейронов), агрессивно мутировать (вероятности 80% и 50%), создавать новые Виды и безжалостно убивать тех, кто не развивается (stagnation). Для CartPole это работает просто идеально!
