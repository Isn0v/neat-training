давай сделаем neat нейросеть для lunar lander

Отличная задача! **Lunar Lander** (Лунный модуль) — это уже серьезное испытание. В отличие от тележки с шестом, здесь гораздо сложнее физика (гравитация, инерция), больше кнопок управления и хитрая система штрафов за трату топлива и жесткие посадки.

### Важная подготовка
Для этой среды Gymnasium использует физический движок **Box2D**. Его нужно установить отдельно. Выполни в консоли:
```bash
pip install "gymnasium[box2d]" neat-python numpy
```

### Что изменится по сравнению с CartPole?
1. **Входы (Inputs): 8 штук.** (Координаты X/Y, скорости X/Y, угол наклона, угловая скорость, касание левой ногой, касание правой ногой).
2. **Выходы (Outputs): 4 штуки.** (0 — ничего не делать, 1 — левый двигатель, 2 — главный нижний двигатель, 3 — правый двигатель).
3. **Условие победы:** Среда считается решенной, если агент набирает **200 очков**.
4. **Сложность:** Задача сложная, поэтому мы увеличим популяцию с 50 до **150 агентов**, а функцию активации сменим с ReLU на **Tanh** (она лучше работает с отрицательными скоростями и координатами).

### Полный код (NEAT + Lunar Lander)

```python
import gymnasium as gym
import neat
import numpy as np
import pickle

# 1. Конфигурация специально под Lunar Lander
CONFIG_TEXT = """
[NEAT]
fitness_criterion      = max
fitness_threshold      = 200
pop_size               = 150
reset_on_extinction    = False
no_fitness_termination = False

[DefaultGenome]
single_structural_mutation = False
structural_mutation_surer  = default

# 8 входов (сенсоры радара и гироскопа), 4 выхода (двигатели)
num_inputs              = 8
num_outputs             = 4
num_hidden              = 0
initial_connection      = full_direct
feed_forward            = True

# Для управления кораблем лучше подходит tanh (выдает от -1 до 1)
activation_default      = tanh
activation_mutate_rate  = 0.1
activation_options      = tanh relu sigmoid

aggregation_default     = sum
aggregation_mutate_rate = 0.0
aggregation_options     = sum

bias_init_type          = gaussian
bias_init_mean          = 0.0
bias_init_stdev         = 1.0
bias_max_value          = 30.0
bias_min_value          = -30.0
bias_mutate_power       = 0.5
bias_mutate_rate        = 0.7
bias_replace_rate       = 0.1

weight_init_type        = gaussian
weight_init_mean        = 0.0
weight_init_stdev       = 1.0
weight_max_value        = 30.0
weight_min_value        = -30.0
weight_mutate_power     = 0.5
weight_mutate_rate      = 0.8
weight_replace_rate     = 0.1

response_init_type      = gaussian
response_init_mean      = 1.0
response_init_stdev     = 0.0
response_max_value      = 30.0
response_min_value      = -30.0
response_mutate_power   = 0.0
response_mutate_rate    = 0.0
response_replace_rate   = 0.0

enabled_default         = True
enabled_mutate_rate     = 0.05
enabled_rate_to_true_add = 0.0
enabled_rate_to_false_add = 0.0

conn_add_prob           = 0.5
conn_delete_prob        = 0.5
node_add_prob           = 0.2
node_delete_prob        = 0.2

compatibility_disjoint_coefficient = 1.0
compatibility_weight_coefficient   = 0.5

[DefaultSpeciesSet]
compatibility_threshold = 3.0

[DefaultStagnation]
species_fitness_func = max
# Даем видам больше времени на развитие, задача сложная
max_stagnation       = 30
species_elitism      = 3

[DefaultReproduction]
elitism            = 2
survival_threshold = 0.2
min_species_size   = 1
"""

def create_config_file(filename="config-lunar.txt"):
    with open(filename, "w") as f:
        f.write(CONFIG_TEXT)
    return filename

def eval_genomes(genomes, config):
    # Создаем среду. Версия v3 актуальна для новых версий Gymnasium
    env = gym.make('LunarLander-v3')
    
    for genome_id, genome in genomes:
        net = neat.nn.FeedForwardNetwork.create(genome, config)
        
        state, info = env.reset()
        fitness = 0.0
        done = False
        truncated = False
        
        while not (done or truncated):
            # Нейросеть выдает 4 числа
            output = net.activate(state)
            
            # Выбираем двигатель, у которого уверенность (число) больше всего
            action = np.argmax(output) 
            
            state, reward, done, truncated, info = env.step(action)
            fitness += reward
            
        genome.fitness = fitness
        
    env.close()

def run_neat():
    config_path = create_config_file()
    
    config = neat.Config(neat.DefaultGenome, neat.DefaultReproduction,
                         neat.DefaultSpeciesSet, neat.DefaultStagnation,
                         config_path)

    p = neat.Population(config)

    p.add_reporter(neat.StdOutReporter(True))
    stats = neat.StatisticsReporter()
    p.add_reporter(stats)

    print("Начинаем тренировку пилотов...")
    # Ставим максимум 200 поколений. Но обычно решает за 50-100.
    winner = p.run(eval_genomes, 200)

    print(f'\nОбучение завершено! Лучший геном:\n{winner}')

    # Сохраняем мозг лучшего пилота в файл, чтобы не обучать заново каждый раз
    with open("best_lunar_pilot.pkl", "wb") as f:
        pickle.dump(winner, f)

    # Демонстрация
    print("\nЗапускаем демонстрацию лучшего пилота...")
    env_eval = gym.make('LunarLander-v3', render_mode='human')
    winner_net = neat.nn.FeedForwardNetwork.create(winner, config)
    
    # Покажем 3 посадки подряд
    for _ in range(3):
        state, info = env_eval.reset()
        done = False
        truncated = False
        total_reward = 0
        
        while not (done or truncated):
            output = winner_net.activate(state)
            action = np.argmax(output)
            state, reward, done, truncated, info = env_eval.step(action)
            total_reward += reward
            
        print(f"Результат посадки: {total_reward:.1f} очков")
        
    env_eval.close()

if __name__ == '__main__':
    run_neat()
```

В `LunarLander` каждый раз, когда мы вызываем `env.reset()`, корабль появляется на разной высоте, с разным углом наклона и с разной начальной скоростью (его как бы "подбрасывает" в случайную сторону). 
Один из твоих мутантов просто идеально заспавнился прямо над посадочной площадкой без боковой скорости. Он просто жал кнопку "газ вниз" и случайно сел, получив 213 очков. Алгоритм NEAT поверил, что этот мутант гений, и остановил обучение.

### Как это исправить?
Чтобы доказать, что пилот реально умеет летать, а не просто везунчик, **каждому геному нужно сдать экзамен несколько раз** (например, посадить корабль в 3 или 5 разных случайных условиях). Итоговой оценкой будет **среднее арифметическое** за все попытки.

Вот как нужно изменить функцию `eval_genomes` (просто замени этот блок в своем коде):

```python
def eval_genomes(genomes, config):
    env = gym.make('LunarLander-v3')
    
    # СКОЛЬКО РАЗ ТЕСТИРУЕМ КАЖДОГО ПИЛОТА
    episodes_per_genome = 3 
    
    for genome_id, genome in genomes:
        net = neat.nn.FeedForwardNetwork.create(genome, config)
        total_fitness = 0.0
        
        # Запускаем серию посадок для одного и того же мозга
        for _ in range(episodes_per_genome):
            state, info = env.reset() # Каждый раз новая случайная генерация старта
            episode_fitness = 0.0
            done = False
            truncated = False
            
            while not (done or truncated):
                output = net.activate(state)
                action = np.argmax(output) 
                state, reward, done, truncated, info = env.step(action)
                episode_fitness += reward
                
            total_fitness += episode_fitness
            
        # Записываем средний результат за все попытки
        genome.fitness = total_fitness / episodes_per_genome
        
    env.close()
```

### Что изменится теперь:
1. Пилот больше не сможет пройти тест за счет удачного старта. Если в первый раз он сел (200 очков), а во второй и третий разбился (-100 и -100), то его средний балл будет `0`. Он не достигнет порога в 200 очков и эволюция продолжится.
2. Итоговая нейросеть будет **железобетонной**. Она научится выравнивать корабль из любого штопора.
3. **Минус:** Обучение замедлится в 3 раза, так как вместо 150 игр за поколение, компьютеру теперь нужно отыграть 450 игр.

### 💡 Про-совет: Как ускорить обучение в несколько раз

Так как игр стало много, Python может начать тормозить, считая их по очереди. `neat-python` умеет использовать **все ядра твоего процессора** одновременно!

Чтобы включить турбо-режим (мультипроцессинг), в основном цикле `run_neat` нужно заменить обычный запуск на `ParallelEvaluator`.

Вот как должен выглядеть конец твоего файла (функция `run_neat`):

```python
def eval_dummy(genome, config):
    # Эта функция нужна для многопоточности, 
    # ParallelEvaluator будет вызывать её для каждого генома отдельно
    env = gym.make('LunarLander-v3')
    episodes_per_genome = 3
    net = neat.nn.FeedForwardNetwork.create(genome, config)
    total_fitness = 0.0
    
    for _ in range(episodes_per_genome):
        state, info = env.reset()
        episode_fitness = 0.0
        done = False
        truncated = False
        while not (done or truncated):
            output = net.activate(state)
            action = np.argmax(output) 
            state, reward, done, truncated, info = env.step(action)
            episode_fitness += reward
        total_fitness += episode_fitness
    env.close()
    return total_fitness / episodes_per_genome


def run_neat():
    config_path = create_config_file()
    config = neat.Config(neat.DefaultGenome, neat.DefaultReproduction,
                         neat.DefaultSpeciesSet, neat.DefaultStagnation,
                         config_path)
    p = neat.Population(config)
    p.add_reporter(neat.StdOutReporter(True))
    stats = neat.StatisticsReporter()
    p.add_reporter(stats)

    print("Начинаем тренировку пилотов (Используем все ядра CPU)...")
    
    # ВАЖНО: 4 - это количество потоков. 
    # Если у тебя 8-ядерный процессор, можешь поставить 8.
    pe = neat.ParallelEvaluator(4, eval_dummy)
    
    # Запускаем эволюцию через ParallelEvaluator
    winner = p.run(pe.evaluate, 200)

    print(f'\nОбучение завершено! Лучший геном:\n{winner}')
    
    # Дальше код демонстрации (env_eval), он остается без изменений
    # ...
```

### Что произошло? (Почему эволюция сделала его тупым)
Если при рождении дать кораблю случайные веса в мозге, он начинает хаотично палить изо всех двигателей. Корабль начинает бешено крутиться, улетает за экран или разбивается вверх ногами. За это игра дает **штраф -300 или -400 очков**.
Но если корабль **вообще ничего не делает**, он просто камнем падает вниз. За это игра дает **штраф -80 очков**.

Эволюция посмотрела на это и решила: *"Ага! Пытаться лететь — это больно (-400). Ничего не делать — это нормально (-80). Значит, выживают те, кто ничего не делает!"*

Отличный вопрос! Чтобы понять эти параметры, нужно вспомнить, как NEAT работает со связями в мозге.

В NEAT связи (синапсы) между нейронами можно не только создавать или удалять навсегда. Их можно временно **«выключать»** (переводить параметр `enabled` в `False`). 
Это похоже на то, как в реальном мозге некоторые нейронные пути засыпают за ненадобностью, но физически остаются на месте. Сигнал по выключенной связи не идет.

Параметры, которые ты добавил, управляют **однонаправленными (асимметричными) мутациями** этого выключателя:

### 1. `enabled_rate_to_true_add = 0.0`
Это вероятность того, что спящая (выключенная) связь при мутации у ребенка **принудительно проснется** и станет активной (`True`).

### 2. `enabled_rate_to_false_add = 0.0`
Это вероятность того, что активная связь принудительно **уснет** и перестанет передавать сигнал (`False`).

---

### А зачем они нужны, если уже есть `enabled_mutate_rate = 0.01`?

Параметр `enabled_mutate_rate = 0.01` (из нашего конфига) работает как **симметричный переключатель**. Он говорит: *"С вероятностью 1% просто инвертируй текущее состояние. Было включено — выключи. Было выключено — включи"*.

А параметры `rate_to_true_add` и `rate_to_false_add` позволяют задать алгоритму **перекос**. 
Например, если бы мы поставили `enabled_rate_to_false_add = 0.2`, то каждое поколение 20% всех рабочих связей в мозге принудительно отключались бы. 

### Почему для нашей задачи мы поставили их в `0.0`?
Помнишь проблему, с которой мы боролись в прошлом сообщении? Твой пилот **сам себе сделал лоботомию**, отключив все связи, лишь бы корабль падал ровно и не тратил топливо.

Установив эти два параметра в ноль (`0.0`), мы строго запрещаем алгоритму массово и целенаправленно «глушить» или «пробуждать» куски мозга. Мы оставили только честный, крошечный шанс в 1% (`enabled_mutate_rate = 0.01`), что случайная связь переключится. Это делает мутации более стабильными, и пилоты перестают "тупеть".

def run_neat():
    config_path = create_config_file()
    config = neat.Config(neat.DefaultGenome, neat.DefaultReproduction,
                         neat.DefaultSpeciesSet, neat.DefaultStagnation,
                         config_path)
    
    p = neat.Population(config)
    p.add_reporter(neat.StdOutReporter(True))
    stats = neat.StatisticsReporter()
    p.add_reporter(stats)

    print("Начинаем тренировку пилотов (Ищем как минимум ТРОИХ победителей)...")
    
    # Создаем многопоточный оценщик
    pe = neat.ParallelEvaluator(4, eval_dummy)
    
    # Сюда мы сохраним нашу тройку (или больше) победителей
    champions_list = []

    # Создаем функцию-обертку прямо внутри run_neat
    def evaluate_and_check(genomes, config):
        # 1. Запускаем стандартную многопоточную оценку. 
        # После этой строки у каждого генома появится свой fitness
        pe.evaluate(genomes, config)
        
        # 2. Считаем, сколько геномов набрали 200 очков или больше
        current_champions = [g for genome_id, g in genomes if g.fitness is not None and g.fitness >= 200]
        
        # 3. Если их 3 или больше — прерываем эволюцию!
        if len(current_champions) >= 3:
            nonlocal champions_list
            champions_list = current_champions
            # Генерируем исключение, чтобы выпрыгнуть из цикла p.run()
            raise StopIteration(f"Успех! Найдено {len(current_champions)} пилотов-профи.")

    # Запускаем эволюцию, обернув её в блок try-except
    try:
        # Обрати внимание: передаем нашу новую функцию evaluate_and_check
        p.run(evaluate_and_check, 200)
    except StopIteration as e:
        print(f"\n[Эволюция досрочно остановлена]: {e}")

    # Подводим итоги
    if champions_list:
        # Сортируем победителей по количеству очков (от большего к меньшему)
        champions_list.sort(key=lambda x: x.fitness, reverse=True)
        winner = champions_list[0] # Берем самого лучшего из сдавших экзамен
        print(f"\nЛучший из команды победителей набрал: {winner.fitness:.1f} очков!")
    else:
        print("\nЗа 200 поколений не удалось найти 3-х идеальных пилотов.")
        winner = p.best_genome # Берем хотя бы того, кто есть

    # Сохраняем мозг лучшего
    with open("best_lunar_pilot.pkl", "wb") as f:
        pickle.dump(winner, f)

    # Демонстрация
    print("\nЗапускаем демонстрацию лучшего пилота из команды...")
    env_eval = gym.make('LunarLander-v3', render_mode='human')
    winner_net = neat.nn.FeedForwardNetwork.create(winner, config)
    
    for _ in range(3):
        state, info = env_eval.reset()
        done = False
        truncated = False
        total_reward = 0
        
        while not (done or truncated):
            output = winner_net.activate(state)
            action = np.argmax(output)
            state, reward, done, truncated, info = env_eval.step(action)
            total_reward += reward
            
        print(f"Результат демонстрационной посадки: {total_reward:.1f} очков")
        
    env_eval.close()

### Как это работает:
1. `pe.evaluate(genomes, config)` берет всех 150 агентов из поколения, раскидывает их по ядрам процессора и играет каждым по 3 игры (как мы сделали раньше).
2. Как только все 150 агентов оценены, выполняется генератор списков (list comprehension): мы пробегаем по всем 150 результатам и забираем тех, у кого оценка `>= 200`.
3. Если длина этого списка `>= 3`, мы кричим интерпретатору Python: `"StopIteration!"`. Это мгновенно обрывает внутренний цикл `neat-python`, не дожидаясь 200-го поколения.
4. В блоке `except StopIteration` мы мягко ловим этот крик, сортируем победителей и забираем абсолютного чемпиона на демонстрацию.

О, это прекрасная ошибка! Она означает, что мы сделали мозг агентов настолько сложным при рождении, что **эволюция сошла с ума от разнообразия**.

### Что именно произошло (математика ошибки):
Помнишь, мы добавили `num_hidden = 4`? Теперь каждый из 150 агентов рождается с уникальным, довольно большим переплетением нейронов. 
Алгоритм NEAT измеряет "генетическое расстояние" между ними. У нас в конфиге стоит строгое правило: `compatibility_threshold = 3.0`.
Из-за новых извилин разница между всеми 150 агентами стала больше, чем 3.0. Поэтому алгоритм сказал: *"Ого, вы все абсолютно разные животные!"* — и **создал 150 разных видов для 150 агентов**.

А дальше сломалась математика:
У алгоритма есть правило `min_species_size = 2` (каждый вид должен оставить минимум 2 потомка, чтобы не вымереть сразу).
150 видов × 2 потомка = **300 детей**. 
А размер популяции у нас жестко ограничен: `pop_size = 150`. Питон понял, что не может впихнуть 300 детей в 150 мест, и выбросил `RuntimeError`.

---

### Как это исправить?

Нам нужно просто **сделать алгоритм более терпимым к разнице в генах**, чтобы он объединял пилотов в группы (виды) по 20-30 штук, а не дробил их по одному. И заодно разрешить видам временно состоять из 1 особи.

Найди в своем `CONFIG_TEXT` эти две секции в самом низу и замени их:

```python
[DefaultSpeciesSet]
# БЫЛО 3.0. Делаем порог выше! Теперь агентам проще считаться "родственниками"
compatibility_threshold = 5.0

[DefaultStagnation]
species_fitness_func = max
max_stagnation       = 30
species_elitism      = 3

[DefaultReproduction]
elitism            = 2
survival_threshold = 0.2
# БЫЛО 2. Ставим 1, чтобы алгоритм никогда больше не падал от нехватки мест
min_species_size   = 1
```

### Что изменится после этого фикса:
Когда ты запустишь код, обрати внимание на строчку в консоли:
`Population of 150 members in X species`
Раньше там X стремилось к 150. С порогом `5.0` видов станет нормальное количество (от 5 до 15). Они будут конкурировать друг с другом группами, как и задумано в NEAT, и ошибка больше никогда не появится!
